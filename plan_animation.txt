ANIMATION SYSTEM PLAN
=====================

OVERVIEW
--------
Replace the scattered shake/fizzle flags in the reducer with a unified,
registry-driven animation system. Cards define what animation to play and
how intense. The registry defines how each animation looks/sounds. BattleScreen
is the only place that manages animation lifecycle.

FILES TO CREATE
---------------
1. src/battle/animationRegistry.js
   - Map of animation name -> { cssClass, duration, sfx, volume }
   - Commented placeholders for future: sprite, particles
   - Initial entries: shake, fizzle

2. src/battle/animations.css
   - All @keyframes definitions
   - All .animate-* classes using var(--anim-intensity, 1)
   - Initial: animate-shake, animate-fizzle

FILES TO MODIFY
---------------
3. src/data/cards/fighter_cards.js
   - Add `animation` field to each card (string or array)
   - Add `animation_intensity` field (float, 1.0 = default)

4. src/battle/engine/battle_engine.js
   - ExecuteAction returns animationHint: action.animation ?? 'shake'
   - ExecuteAction returns animationIntensity: action.animation_intensity ?? 1.0

5. src/battle/reducer.js
   - BATTLE_STEP: replace shakingEnemyId, shakingPlayerId, fizzlingCard
     with pendingAnimation: { type, targetId, intensity }
   - Remove STOP_SHAKE, STOP_PLAYER_SHAKE, STOP_FIZZLE cases

6. src/screens/BattleScreen.jsx
   - Add activeAnimations local state: { [targetId]: { cssClass, intensity } }
   - Replace 3 animation useEffects with 1 that reads pendingAnimation,
     looks up registry, sets activeAnimations, plays SFX, auto-clears
   - Import animations.css

7. src/components/battle/EnemyZone.jsx
   - Replace isShaking check with activeAnimations[enemy.id]
   - Pass animClass + set --anim-intensity CSS variable on element

8. src/components/battle/PlayerPortrait.jsx
   - Same as EnemyZone

INTENSITY SYSTEM
----------------
- Cards carry animation_intensity (e.g. 1.0, 1.5, 2.0)
- Passed through reducer as pendingAnimation.intensity
- BattleScreen sets CSS custom property: style={{ '--anim-intensity': intensity }}
- animations.css uses var(--anim-intensity, 1) to scale speed, brightness, etc.

FUTURE PLACEHOLDERS (not implemented now)
------------------------------------------
- sprite: null   // { sheet, frames, fps } for sprite sheet animations
- particles: null // { preset } for particle burst effects

REMOVALS
--------
- shakingEnemyId from reducer state
- shakingPlayerId from reducer state
- fizzlingCard from reducer state
- STOP_SHAKE reducer case
- STOP_PLAYER_SHAKE reducer case
- STOP_FIZZLE reducer case
- 3 separate animation useEffects in BattleScreen

EASY REMOVAL
------------
To disable all animations: comment out setActiveAnimations() call in BattleScreen useEffect.
To remove the system entirely: delete registry + css, remove the one useEffect,
remove activeAnimations state, remove animClass from components.
