--------------------------------------------------------------------------------
  PROBLEMS WITH CURRENT STRUCTURE
--------------------------------------------------------------------------------

1. components/ is flat — all battle UI components sit in the same folder.
   Adding Title, Map, CharacterSelect, PlayerMenu screens will make this unmanageable.

2. App.jsx does too much — state, phase routing, battle loop, and JSX all in one file.
   Needs to be split as new screens are added.

3. asssets/ has a typo — should be assets/ (single 's').

4. No screens/ layer — nowhere to put TitleScreen, MapScreen, CharacterSelectScreen, etc.

5. Loose untracked Upscaled_*.png files in asssets/ root — unorganized.

--------------------------------------------------------------------------------
  PHASE ROUTING IN App.jsx (after restructure)
--------------------------------------------------------------------------------

  TITLE → CLASS_SELECT → MAP → QUEUE_SETUP → BATTLE → RESULT

  App.jsx becomes a simple switch:

    switch (state.phase) {
      case 'TITLE':           return <TitleScreen />
      case 'CLASS_SELECT':    return <CharacterSelectScreen />
      case 'MAP':             return <MapScreen />
      case 'QUEUE_SETUP':
      case 'BATTLE':          return <BattleScreen />
      case 'RESULT':          return <ResultScreen />
    }
